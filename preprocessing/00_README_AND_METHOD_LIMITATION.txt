RNA-seq pre-heatmap workflow output

Input files
-----------
- raw_counts_all_35_ordered(1).csv
- sample_metadata_all_35_ordered(1).csv
- target_gene_annotation_167(1).csv

Cohort order
------------
Normal -> Obese -> Diabetic -> HFrEF
D8S was handled as Diabetic.

Implemented operations
----------------------
1. Input validation and metadata/order reconciliation.
2. edgeR-style filterByExpr logic based on the no-intercept cohort design.
3. TMM normalization computed from filtered counts and library sizes recalculated after filtering (edgeR `keep.lib.sizes=FALSE` convention).
4. Voom-scale log2-CPM transformation and mean-variance precision weights.
5. All 35 samples retained; visual QC flags are review flags only.
6. Target-gene extraction after filtering/TMM/voom and Normal-mean centering.

Implementation limitation
-------------------------
The execution environment lacked R, edgeR, and limma. Accordingly, the exported files labelled METHOD_MATCHED_PROVISIONAL were produced by an auditable Python implementation of the requested mathematical workflow, including TMM, voom-style log2-CPM/weights, WLS cohort means, and moment-based empirical-Bayes variance moderation. Use the supplied `08_exact_edgeR_limma_voom_R_workflow.R` in R with edgeR and limma for final formal DE claims or publication-grade finalization. The target centered matrix is suitable for review of values and ordering but should be regenerated by the R script for a final report.

Important heatmap rule retained
-------------------------------
No heatmap was generated. For every target gene and every individual, centered expression equals that individual's voom-scale log2-CPM minus the mean voom-scale log2-CPM of the nine Normal samples for the same gene. Normal individual columns remain distinct and vary around zero.
